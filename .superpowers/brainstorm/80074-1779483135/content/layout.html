Proposed Page & Wrapper Architecture

Here is the visual mapping of the Barba.js wrapper structure, capsule navbar states, and active link syncing mechanism.

1. DOM Wrapper Architecture

This wrapper layout is required so Barba.js can replace content inside the <main> tag, while the navigation and transition screen remain persistent and immune to reloads.

<body data-barba="wrapper"> (Global Wrapper / Event Hub)
<nav class="nav"> (Persistent Navbar - Sits outside the page container)
Holds logo transition, links, dropdown menu, and marquee bar.
<div class="transition"> (Persistent Fullscreen Page-Wipe Transition Screen)
Opacity: 0, Pointer-events: none. Opacity fades in to cover the screen during page load.
<main class="main" data-barba="container" data-barba-namespace="home"> (Swapped Content)
Everything inside here is replaced by Barba on page transitions.
<div class="under-nav-bar"> <div class="under-nav-bar__inner"> ... page content ... </div> </div>

4. Active Link Update Mechanism

When Barba swaps the page, we parse the incoming document, extract links marked with [data-barba-update], and sync their attributes:

// Before Sync (On "About" page, but Nav links are from "Home" state):
<a href="/about" data-barba-update>About</a> (Normal state)

// After Sync (Synced during Barba hooks enter):
<a href="/about" data-barba-update class="nav-bar__link is--active" aria-current="page">About</a> (Active state applied dynamically)
