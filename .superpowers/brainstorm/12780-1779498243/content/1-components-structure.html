Webflow Component Architecture

Let's refine the component structure, properties, and slots before porting them to Webflow.

1. Navbar Link Component: nav-bar__big-li

To replicate the menu links cleanly in Webflow, we can create a single Navbar Link component with these properties:

Component Schema & Properties:

Property Name Type Description Example

2. Marquee Component Strategy (Visibility and Placement)

The green infinite marquee sits below the navbar, but how should we manage its presence across pages?

A

Page Layout Slot Strategy (Recommended)

Keep the marquee outside the navbar component. Pages that need it simply place the Marquee component directly inside their page wrapper. In pages that do NOT require the marquee, it is completely omitted from the layout. This avoids loading unused DOM and simplifies page-specific styling.

B

Marquee Component Visibility Properties

Make the marquee a direct child of the navbar or page template, and control its visibility using a Webflow component property (e.g. show-marquee = false) or custom utility classes (like a hide/show variant). Keeps the navbar + marquee packaged together, but increases component complexity.

3. Persistent vs Non-Persistent Structure

How page sections should be split for Barba PJAX swaps and Webflow template clean structure:

Global / Persistent (Never Swapped)
  • Page Transitions Screen (Wipe overlays)
  • Persistent Capsule Navbar (Stays open/closed during PJAX swaps)
  • Global Custom Code Embeds (Lenis init, page-swipe listeners)
Page / Non-Persistent (Swapped by PJAX)
  • Infinite Marquee Component (Allows page-specific marquee text / color themes!)
  • Main Page Body Container (Hero, sections, footer)
  • Page-specific Scripts or Custom SEO Tags
